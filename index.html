01 Base para el Equipo
Instalador Base (Homebrew)
PASO 1 ⌨ Terminal
Esta es la herramienta necesaria para que el Mac pueda cargar programas profesionales. Solo cópiela y ejecútela una vez.
Terminal
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Nota: Tras terminar, si el Terminal muestra instrucciones de "Next Steps", cópielas y ejecútelas también.
Perfil de Andrés (Git)
PASO 2 ⌨ Terminal
Establece el nombre y correo de Andrés en el sistema. Copie ambos pasos por separado.
Terminal
1️⃣ Paso 1: brew install git
2️⃣ Paso 2: git config --global user.name "Andrés Cebay"
3️⃣ Paso 3: git config --global user.email "cebayy044@gmail.com"
Terminal Pro (Oh My Zsh + Plugins)
⌨ Terminal Productividad
Optimiza tu flujo de trabajo con autocompletado inteligente y colores en la terminal.
Terminal
1️⃣ Oh My Zsh: sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
2️⃣ Autosuggestions: git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
3️⃣ Syntax Highlighting: git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting
4️⃣ Activar: sed -i '' 's/plugins=(git)/plugins=(git zsh-autosuggestions zsh-syntax-highlighting)/' ~/.zshrc && source ~/.zshrc

03 Stack PHP / Laravel
PHP 8.3 + Composer
Obligatorio ⌨ Terminal
Necesario para que el Mac entienda el código PHP de Andrés. Copie y ejecute estos pasos en orden.
Terminal
1️⃣ Paso 1: brew install php@8.3
2️⃣ Paso 2: echo 'export PATH="/opt/homebrew/opt/php@8.3/bin:$PATH"' >> ~/.zshrc
3️⃣ Paso 3: echo 'export PATH="/opt/homebrew/opt/php@8.3/sbin:$PATH"' >> ~/.zshrc
4️⃣ Paso 4: source ~/.zshrc
Herramientas Adicionales
5️⃣ Paso 5: brew install composer
6️⃣ Paso 6: composer global require laravel/installer
7️⃣ Paso 7: echo 'export PATH="$HOME/.composer/vendor/bin:$PATH"' >> ~/.zshrc
8️⃣ Paso 8: source ~/.zshrc
Verificar
php -v
composer --version
laravel --version

Obligatorio ↓ Descarga directa

↗ Descargar Laravel Herd (Gratis)

Bases de Datos (DBngin)
OBLIGATORIO ↓ Descarga visual

PostgreSQL (Base de Datos Avanzada)
Opcional ⌨ Terminal
Si el proyecto de Andrés usa PostgreSQL en lugar de MySQL (Postgres), instálelo con estos comandos:
Terminal
1️⃣ Instalar: brew install postgresql@16
2️⃣ Iniciar: brew services start postgresql@16
Configuración .env en Laravel:
DB_CONNECTION=pgsql DB_HOST=127.0.0.1 DB_PORT=5432 DB_DATABASE=nombre_db DB_USERNAME=postgres DB_PASSWORD=
Manejo de DB (DBeaver community)
Dev ↓ Descarga directa
El "Swiss Army Knife" para bases de datos profesional. Permite visualizar y editar PostgreSQL de forma visual.
↗ Descargar DBeaver para Mac
Cómo conectar a Postgres: Seleccione "PostgreSQL" -> Host: 127.0.0.1 -> Puerto: 5432 -> Usuario: postgres.
04 JavaScript / Frontend
Node.js + npm
Dev ⌨ Terminal
Herramientas para manejar el diseño y comportamiento de las webs. Siga los pasos numerados.
Terminal
1️⃣ Paso 1: brew install nvm
2️⃣ Paso 2: mkdir -p ~/.nvm
3️⃣ Paso 3: echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc
4️⃣ Paso 4: [ -s "$HOMEBREW_PREFIX/opt/nvm/nvm.sh" ] && \. "$HOMEBREW_PREFIX/opt/nvm/nvm.sh"
5️⃣ Paso 5: source ~/.zshrc
Configuración de Node
6️⃣ Paso 6: nvm install --lts
7️⃣ Paso 7: nvm use --lts
05 Laravel + React (Vite)
Instalación de React en Laravel
Front ⌨ Terminal
Agregue las librerías necesarias para que React funcione dentro de su proyecto de Laravel.
Terminal
1️⃣ Paso 1: npm install @vitejs/plugin-react react react-dom
Preferencia rápida: Si prefiere el paquete oficial con login y todo listo, use: composer require laravel/breeze --dev y luego php artisan breeze:install react.
Configuración de Vite (React Plugin)
Config
Modifique su archivo vite.config.js para incluir el plugin de React.
import react from '@vitejs/plugin-react'; // En plugins: [ ..., react() ]
En su archivo Blade principal (layout), agregue estas directivas:
@viteReactRefresh @vite(['resources/js/app.jsx'])

07 IA Local & Agentes
Ollama (Cerebro Local)
IA ⌨ Terminal
Esta es la herramienta principal para correr IAs en tu propio Mac sin depender de la nube. Al ser un M1 Max, funcionará muy rápido.
Terminal
brew install --cask ollama
Después de instalar, abre la aplicación "Ollama" desde tu carpeta de aplicaciones para iniciar el servidor.
Modelos IA Locales (Llama, Qwen, DeepSeek)
IA ⌨ Terminal
Descargue estos modelos recomendados para desarrollo. Se descargarán automáticamente la primera vez que los corra.
Terminal
1️⃣ Llama 3.1 (8B): ollama run llama3.1:8b
2️⃣ Qwen 2.5 Coder (32B): ollama run qwen2.5-coder:32b
3️⃣ DeepSeek Coder V2 (16B): ollama run deepseek-coder-v2:16b
¿Cómo correrlas después? Simplemente use el comando ollama run [nombre-modelo] en el terminal en cualquier momento.
Control de Agentes (Claude + Kimi)
IA ⌨ Terminal
Este comando especial permite conectar modelos en la nube como Kimi a través de la interfaz de Claude.
Terminal
ollama launch claude --model kimi-k2.5:cloud
08 Flujo de Trabajo con IA
Contexto de Proyecto (claude.md)
IA Recomendado
Un archivo claude.md en la raíz de su proyecto permite que la IA (Claude, Antigravity) entienda todo el proyecto al instante sin que usted tenga que explicárselo.
Ejemplo de claude.md
# Proyecto Don Camilo Stack: Laravel 11, React (Vite), PostgreSQL. Reglas: Usar Tailwind v4, mantener controladores limpios.
Simplemente cree el archivo en su proyecto y la IA lo leerá automáticamente.

10 Producción & Despliegue (Hostinger)
Preparación de Laravel para Hostinger
Producción ⌨ Terminal
Pasos críticos para que tu sitio funcione en el servidor real sin errores (500). Ejecuta estos comandos en la consola de Hostinger:
Terminal de Hostinger
1️⃣ Permisos: chmod -R 775 storage bootstrap/cache
2️⃣ Optimizar: php artisan config:cache && php artisan route:cache && php artisan view:cache
Recordatorio: No olvides configurar las variables del archivo .env directamente en el Panel de Control de Hostinger.
